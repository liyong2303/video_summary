**5. TaskService.java** at `service/TaskService.java`:
- `@Service` class with `@Slf4j`
- Inject `TaskMapper`, `BilibiliVideoService` (from Plan 02)

- Method `String parseBvid(String url)` (static utility, implements BILI-01):
  - If input matches regex `BV[A-Za-z0-9]+` exactly, return it
  - If input contains `bilibili.com/video/(BV[A-Za-z0-9]+)`, extract and return the BV号
  - If input contains `b23.tv/` (short URL), log a warning that short URLs are not supported yet and throw `IllegalArgumentException` with message "暂不支持b23.tv短链接，请使用完整BV号"
  - If no BV号 can be extracted, throw `IllegalArgumentException` with message "无法识别的视频链接，请输入BV号或B站视频链接"

- Method `SubmitResponse submit(String url)`:
  1. Call `parseBvid(url)` to get bvid
  2. Query existing task: `taskMapper.selectOne(new LambdaQueryWrapper<Task>().eq(Task::getUserId, 0L).eq(Task::getBvid, bvid))`
  3. If found: return `SubmitResponse.builder().taskId(found.getId()).bvid(bvid).videoTitle(found.getVideoTitle()).videoDuration(found.getVideoDuration()).coverUrl(found.getCoverUrl()).status(found.getStatus()).isExisting(true).build()`
  4. If not found:
    a. Call `bilibiliVideoService.getVideoInfo(bvid)` to get video info
    b. Call `bilibiliVideoService.validateDuration(videoInfo.getDuration())` to check duration limit (BILI-04)
    c. Create new Task entity: `Task.builder().userId(0L).bvid(bvid).videoTitle(videoInfo.getTitle()).videoDuration(videoInfo.getDuration()).coverUrl(videoInfo.getCoverUrl()).status(Task.Status.PENDING).build()`
    d. Insert with `taskMapper.insert(task)`
    e. Return `SubmitResponse` with `isExisting(false)`

- Method `TaskResponse getTask(Long taskId)`:
  1. Query task by id: `taskMapper.selectById(taskId)`
  2. If not found, throw `IllegalArgumentException` with message "任务不存在"
  3. Build and return `TaskResponse` mapping all fields

- Method `void processTask(Long taskId)`:
  1. Update status to `PROCESSING`: `task.setStatus(Task.Status.PROCESSING); taskMapper.updateById(task)`
  2. Try:
    a. Get video info and extract subtitles via `bilibiliVideoService`
    b. Call `bilibiliVideoService.extractSubtitles(task.getBvid(), videoInfo.getCid())` — need to store cid in Task entity or re-fetch video info
    c. For this phase: store subtitle text directly in `subtitleStoragePath` field (as a file path or the text itself — use a simple file storage approach: write to `data/subtitles/{bvid}.txt` relative to working directory, store the path in the field)
    d. Update status to `COMPLETED`, set `completedAt`
  3. Catch `SubtitleNotFoundException` -> set status to `FAILED`, set `errorMessage` to the exception message
  4. Catch other exceptions -> set status to `FAILED`, set `errorMessage` to "处理失败：" + exception.getMessage()
  5. Finally: `taskMapper.updateById(task)`

Note: Since this phase does not yet have the AI pipeline, `processTask` will extract subtitles and mark as completed immediately. The AI pipeline processing is Phase 2. For now, the submit endpoint should synchronously extract subtitles and return the result.

Also: the Task entity needs a `cid` field. Add `cid` (Long) field to Task.java and update schema.sql to include `cid BIGINT DEFAULT NULL COMMENT '视频CID'` after `bvid`.

- `POST /submit`:
  ```java
  @PostMapping("/submit")
  public ApiResult<SubmitResponse> submit(@RequestBody @Valid SubmitRequest request) {
      SubmitResponse response = taskService.submit(request.getUrl());
      // After creating the task, process it synchronously (Phase 1 only)
      if (!response.getIsExisting()) {
          taskService.processTask(response.getTaskId());
          // Re-fetch to get updated status
          TaskResponse updated = taskService.getTask(response.getTaskId());
          response.setStatus(updated.getStatus());
          response.setVideoTitle(updated.getVideoTitle());
      }
      return ApiResult.success(response);
  }
  ```

- `GET /{taskId}`:
  ```java
  @GetMapping("/{taskId}")
  public ApiResult<TaskResponse> getTask(@PathVariable Long taskId) {
      return ApiResult.success(taskService.getTask(taskId));
  }
  ```
