**1. RestTemplateConfig.java** at `config/RestTemplateConfig.java`:

**2. WbiKeyService.java** at `bilibili/WbiKeyService.java`:
- `@Service` class that fetches and caches wbi keys (img_key, sub_key) from Bilibili's nav API
- Method `WbiKeys getWbiKeys()`: calls `https://api.bilibili.com/x/web-interface/nav` with the Bilibili cookie from `application.yml` (injected via `@Value("${app.bilibili.cookie}")`), parses the response to extract `wbi_img.img_url` and `wbi_img.sub_url`, strips the URL prefix to get just the key part (remove `https://i0.hdslb.com/bfs/wbi/` prefix and file extension `.png`)
- Cache keys in memory with a `volatile` field, refresh if older than 10 minutes (use `System.currentTimeMillis()` for age check). Add a `@Scheduled(fixedRate = 600000)` method to refresh proactively.
- Inner record/class `WbiKeys(String imgKey, String subKey)`
- Use `@Slf4j` for logging key refresh events

**3. WbiSignService.java** at `bilibili/WbiSignService.java`:
- `@Service` class — this is the ISOLATED wbi signing module per BILI-06
- Method `Map<String, String> sign(Map<String, String> params)`: Takes a map of request parameters and returns a new map with `w_rid` and `wts` added
- Signing algorithm (wbi签名):
  1. Get current `wts` = `System.currentTimeMillis() / 1000`
  2. Add `wts` to the params map
  3. Get `imgKey` and `subKey` from `WbiKeyService.getWbiKeys()`
  4. Concatenate: `mixinKey = imgKey + subKey`
  5. Apply the mixin key table (a fixed 32-element int array that reorders the 64-char concatenated key): extract 32 chars from mixinKey using the positions `[46,47,18,2,53,8,23,32,15,50,10,31,58,3,45,35,27,43,5,49,33,9,42,19,29,28,14,39,12,38,41,16]`
  6. Sort all params by key alphabetically
  7. URL-encode values, concatenate as `key=value&key=value`
  8. Append the 32-char extracted key
  9. Calculate MD5 hash of the concatenated string → this is `w_rid`
  10. Return params map with `w_rid` and `wts` added
- Add `@Slf4j` and log signing events at debug level
- Add a comment at the top: "wbi签名算法 - B站内部接口签名，可能随API更新而变更，本模块独立便于快速适配"

**4. BilibiliApiClient.java** at `bilibili/BilibiliApiClient.java`:
- `@Service` class with `@Slf4j`
- Inject `RestTemplate`, `WbiSignService`, and `@Value("${app.bilibili.cookie}") String cookie`
- Method `JSONObject getVideoInfo(String bvid)`:
  - Call `GET https://api.bilibili.com/x/web-interface/view?bvid={bvid}`
  - Set Cookie header with the configured bilibili cookie
  - Parse JSON response, check `code == 0`, return `data` object
  - Throw custom `BilibiliApiException` if code != 0 with the `message` field
- Method `JSONObject getPlayerInfo(String bvid, Long cid)`:
  - Build params map: `bvid=bvid`, `cid=cid`, `qn=64`, `fnver=0`, `fnval=16`, `fourk=1`
  - Call `WbiSignService.sign(params)` to add wbi signature
  - Call `GET https://api.bilibili.com/x/player/wbi/v2` with signed params
  - Set Cookie header with the configured bilibili cookie
  - Parse JSON response, check `code == 0`, return `data` object
  - Throw `BilibiliApiException` if code != 0
- Create `BilibiliApiException` as a simple RuntimeException subclass in `bilibili/BilibiliApiException.java` with constructor accepting message and cause
- Use `hutool.json.JSONObject` or `com.alibaba.fastjson2.JSONObject` for JSON parsing (add fastjson2 to pom.xml if hutool is insufficient — prefer hutool's JSONUtil since hutool-all is already in pom.xml)

**4. BilibiliVideoService.java** at `bilibili/BilibiliVideoService.java`:
- `@Service` class with `@Slf4j`
- Inject `BilibiliApiClient`, and `@Value("${app.video.duration-limit-minutes:30}") Integer durationLimitMinutes`
- Method `VideoInfo getVideoInfo(String bvid)`:
  1. Call `bilibiliApiClient.getVideoInfo(bvid)`
  2. Parse response: extract `title`, `duration` (in seconds), `pic` (cover URL), `cid` (first page cid from `pages[0].cid`), `owner.name`
  3. Build and return `VideoInfo` object
- Method `List<SubtitleContent> extractSubtitles(String bvid, Long cid)`:
  1. Call `bilibiliApiClient.getPlayerInfo(bvid, cid)`
  2. Parse `subtitle.subtitles` array from response — each item has `lang`, `lang_key`, `subtitle_url`
  3. If subtitles array is empty or null, throw `SubtitleNotFoundException` with message "该视频没有字幕，目前仅支持有字幕的视频"
  4. For the first Chinese subtitle (lang_key contains "zh"), fetch the subtitle JSON URL (it starts with `https://` or `http://` — if it starts with `//`, prepend `https:`)
  5. Parse subtitle JSON: it's an array of `{"from": 0.0, "to": 5.0, "content": "text"}` objects
  6. Return `List<SubtitleContent>`
- Method `void validateDuration(Integer durationSeconds)`:
  1. If `durationSeconds > durationLimitMinutes * 60`, throw `VideoTooLongException` with message "视频时长超过${durationLimitMinutes}分钟限制，免费用户仅支持${durationLimitMinutes}分钟以内的视频"
- Method `String extractSubtitleText(List<SubtitleContent> subtitles)`:
  1. Concatenate all `content` fields with newline separator
  2. Return the full subtitle text string
- Create `SubtitleNotFoundException` at `bilibili/SubtitleNotFoundException.java` extending `RuntimeException`
- Create `VideoTooLongException` at `bilibili/VideoTooLongException.java` extending `RuntimeException`

Note: The subtitle URL from B站 API returns JSON with `body` array containing `{from, to, content}` objects. Use RestTemplate to fetch and parse this URL.
