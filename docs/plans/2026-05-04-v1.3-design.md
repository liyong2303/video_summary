# v1.3 设计方案：提升使用效率

**创建时间：** 2026-05-04
**版本：** v1.3
**目标：** 提升用户使用效率，通过批量处理、模板管理、一键操作减少重复操作

## 一、功能概述

v1.3 版本专注于提升使用效率，包含三个核心功能：
1. **批量处理** - 一次处理多个视频
2. **模板管理** - 保存和复用常用配置
3. **一键操作** - 自定义操作流程并一键执行

## 二、功能模块设计

### 2.1 批量处理功能

**需求等级：** A1（简单版）

**功能描述：**
用户可以一次性提交3-5个B站视频链接，系统串行处理每个视频，全部完成后统一通知。

**UI设计：**
- 在提交页面增加"批量处理"标签页
- 使用Tag式输入框：输入链接后回车，链接显示为可删除的标签
- 限制最多5个视频链接
- 显示"批量处理"按钮
- 处理时显示进度：已完成X/5个
- 完成后显示结果摘要

**技术实现：**
- 前端：使用Element Plus的el-select或el-tag实现Tag输入
- 后端：新增批量提交接口 `POST /api/video/batch-submit`
- 处理逻辑：循环调用单个视频的处理流程
- 通知机制：全部完成后通过SSE或轮询通知用户

**数据流程：**
```
用户输入多个链接 → 点击批量处理 → 后端创建多个任务 → 串行执行 → 完成通知
```

**接口设计：**
```
POST /api/video/batch-submit
Request: { urls: ["BV1xx411c7mD", "BV1yy411c7mE"], style, length }
Response: { batchId, taskIds, status: "processing" }
```

### 2.2 模板管理功能

**需求等级：** B3（个人高级模板）

**功能描述：**
用户可以创建高级模板，包含完整的配置和自定义Prompt，支持分类管理。

**模板包含字段：**
- 模板名称
- 分类标签（如：学习笔记、营销文案、学术研究）
- 生成风格（academic/casual/concise）
- 内容长度（short/standard/long）
- 输出类型列表（summary/article/card/xiaohongshu）
- 关联的自定义Prompt（完整保存Prompt内容）

**UI设计：**
- 新增"模板管理"页面（`/templates`）
- 左侧：模板分类树
- 右侧：模板列表（卡片形式）
- 创建/编辑模板：复选框选择输出类型，关联自定义Prompt
- 应用模板：在提交页面"快速应用模板"下拉框

**技术实现：**
- 新增实体：Template（模板）、TemplateCategory（模板分类）
- 新增接口：模板CRUD + 应用模板接口
- 前端：TemplateView.vue 组件

**数据流程：**
```
创建模板 → 保存完整配置 → 用户选择模板 → 加载配置到提交页面 → 应用配置
```

**接口设计：**
```
GET /api/templates - 获取用户所有模板
POST /api/templates - 创建模板
PUT /api/templates/{id} - 更新模板
DELETE /api/templates/{id} - 删除模板
GET /api/template-categories - 获取分类列表
POST /api/templates/{id}/apply - 应用模板（返回模板配置）
```

### 2.3 一键操作功能

**需求等级：** B1（自定义快捷动作）

**功能描述：**
用户可以配置一系列操作保存为"快捷动作"，一键应用到单个或多个视频。

**支持的原子操作：**
- 重新生成所有内容
- 重新生成指定类型（如：仅重新生成文章）
- 导出Markdown文件
- 导出PDF文件
- 导出Word文件
- 复制摘要到剪贴板
- 复制指定内容到剪贴板
- 复制到社交媒体（小红书/微博/朋友圈，生成文案链接）

**快捷动作配置：**
- 动作名称
- 操作列表（按顺序执行）
- 应用范围（单个视频/批量）

**UI设计：**
- 在提交页面和历史记录页面添加"快捷操作"按钮
- 新增"快捷操作管理"页面（`/quick-actions`）
- 在视频结果页面显示可用的快捷操作按钮
- 执行时显示进度提示

**技术实现：**
- 新增实体：QuickAction（快捷动作）、QuickActionStep（操作步骤）
- 新增接口：快捷动作CRUD、执行快捷动作
- 前端：QuickActionView.vue 组件
- 后端：执行引擎，按顺序执行操作

**数据流程：**
```
创建快捷动作 → 保存操作列表 → 用户点击快捷按钮 → 后端执行操作 → 返回结果
```

**示例快捷动作：**
- "完整再生产"：重新生成所有内容 → 导出PDF → 复制摘要
- "快速发布"：复制小红书文案 → 生成分享链接
- "学习资料包"：重新生成文章 → 导出Word → 导出学习卡片

**接口设计：**
```
GET /api/quick-actions - 获取所有快捷操作
POST /api/quick-actions - 创建快捷操作
PUT /api/quick-actions/{id} - 更新快捷操作
DELETE /api/quick-actions/{id} - 删除快捷操作
POST /api/quick-actions/{id}/execute - 执行快捷操作
```

## 三、数据库设计

### 3.1 模板分类表（template_category）
```sql
CREATE TABLE template_category (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT NOT NULL,
    name VARCHAR(100) NOT NULL,
    sort_order INT DEFAULT 0,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_user_id (user_id)
)
```

### 3.2 模板表（template）
```sql
CREATE TABLE template (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT NOT NULL,
    category_id BIGINT,
    name VARCHAR(200) NOT NULL,
    style VARCHAR(20) NOT NULL DEFAULT 'concise',
    length VARCHAR(20) NOT NULL DEFAULT 'standard',
    output_types JSON NOT NULL,
    custom_prompt_ids JSON,  -- 关联的自定义Prompt ID列表
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_user_id (user_id),
    INDEX idx_category_id (category_id)
)
```

### 3.3 快捷操作表（quick_action）
```sql
CREATE TABLE quick_action (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT NOT NULL,
    name VARCHAR(200) NOT NULL,
    steps JSON NOT NULL,  -- 操作步骤列表
    apply_scope VARCHAR(20) NOT NULL DEFAULT 'single',  -- single/batch
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_user_id (user_id)
)
```

**steps JSON 结构示例：**
```json
[
  {
    "type": "regenerate",
    "outputType": "all"
  },
  {
    "type": "export",
    "format": "pdf"
  },
  {
    "type": "copy",
    "content": "summary"
  }
]
```

## 五、技术架构

### 5.1 前端架构
- **组件复用：** 提取Tag输入、操作选择等通用组件
- **状态管理：** 使用Pinia管理模板、快捷操作状态
- **路由导航：** 在App.vue添加"模板管理"、"快捷操作"导航

### 5.2 后端架构
- **服务层：** 新增TemplateService、QuickActionService
- **控制器：** 新增TemplateController、QuickActionController
- **执行引擎：** QuickActionExecutor 负责执行快捷操作
- **类型处理：** 使用现有的JsonTypeHandler处理JSON字段

### 5.3 数据流
```
前端页面 → API接口 → Service层 → Mapper层 → 数据库
                      ↓
                  执行引擎 → 其他服务（TaskService等）
```

## 八、风险评估

### 8.1 技术风险
- **批量处理性能：** 串行处理5个视频可能耗时较长
  - 缓解：显示详细进度，支持取消
- **JSON字段存储：** 自定义Prompt内容较大，存储在JSON字段可能影响性能
  - 缓解：考虑使用关联表存储，JSON仅存储ID列表

### 8.2 用户体验风险
- **功能复杂度：** 三个新功能可能增加学习成本
  - 缓解：提供引导教程和默认模板/快捷操作
- **批量处理失败：** 某个视频失败可能影响整体体验
  - 缓解：标记失败视频，继续处理其他视频

## 九、后续优化方向

1. **批量处理并行化**：支持并发处理多个视频
2. **模板分享**：支持用户间分享模板
3. **快捷操作条件判断**：支持复杂的工作流配置
4. **模板市场**：官方预设模板和社区分享
5. **操作历史**：记录快捷操作执行历史
