# VideoSum 版本路线图

> 制定日期：2026-05-01  
> 项目定位：小范围分享（朋友/团队内部使用）  
> 规划思路：用户价值驱动 —— 每个版本交付用户可感知的完整体验升级

---

## 总览

```
v1.0  能用版   ── 补齐骨架，正式可上线
v1.1  好用版   ── 内容可编辑，体验更顺滑
v1.2  创作版   ── Prompt 可定制，内容更个性
v1.3  分发版   ── 一键推送到飞书、Notion 等平台
──────────────────────────────────────────────
v2.0  架构版   ── 插件化重构：视频源/管线/分发三层解耦
v2.1  多平台版 ── 在新架构上接入 YouTube、本地字幕等
v2.2  协作版   ── 团队工作区、共享模板、协作编辑
```

**架构边界说明：**  
v1.x 在现有三服务架构（Vue + Spring Boot + FastAPI）内做功能增量；  
v2.0 是一次架构重构，将视频源、AI 管线、分发目标三层插件化，使系统从"面向B站的单体工具"升级为"可扩展的视频内容处理平台"。

---

## v1.0 — 能用版

**版本目标：** 所有功能端到端跑通，小伙伴能注册登录独立使用。

### 待办清单

| 编号 | 功能 | 说明 |
|------|------|------|
| AUTH-01 | 用户注册接口 | `POST /api/auth/register`，写入 user 表 |
| AUTH-02 | 用户登录 + JWT | `POST /api/auth/login`，Sa-Token 签发 token |
| AUTH-03 | 会话持久化 | 前端存储 token，刷新页面保持登录状态 |
| AUTH-04 | 当前用户接口 | `GET /api/auth/me`，返回用户名 + 今日用量 |
| QUOTA-01 | 每日3次限制 | 免费用户提交前检查 daily_usage 表 |
| QUOTA-02 | 提交时扣次数 | 任务创建成功后计数 +1，超限返回 429 |
| QUOTA-03 | 每日0点重置 | Spring `@Scheduled` 清空 daily_usage |
| QUOTA-04 | 前端额度展示 | 导航栏显示"今日剩余 X 次" |
| FIX-01 | 重新生成接口 | Python 服务补充 `/pipeline/execute-single` 路由 |
| CFG-01 | 部署配置分离 | application.yml 区分 dev/prod profile，敏感项用环境变量注入 |

**交付标准：** 可以把部署地址发给朋友，朋友能注册账号、提交视频、查看历史、重新生成内容。

## v2.0 — 架构版（重构里程碑）

**版本目标：** 架构重构，将视频源、AI 管线、分发目标三层插件化，为后续多平台扩展打基础。

### 架构背景

当前架构的三个硬限制：

| 问题 | 当前状态 | 影响 |
|------|----------|------|
| 视频来源硬编码 | B站抓取写死在 Java 后端 | 加 YouTube 需改核心服务 |
| AI 管线固定拓扑 | 4步骤写死在 `pipeline.py` | 无法调整步骤顺序或新增步骤 |
| 输出目标单一 | 结果只存 MySQL | 每接一个分发平台都要硬编码 |

### 核心架构升级

#### ① 视频源适配器层（Source Adapter）

```
VideoSourceAdapter (interface)
  ├── BilibiliAdapter    # 现有逻辑迁移
  ├── YouTubeAdapter     # v2.1 接入
  ├── LocalFileAdapter   # v2.1 接入（上传字幕文件）
  └── DouyinAdapter      # 未来扩展
```

每个 Adapter 实现统一接口：`extractSubtitle(url) → SubtitleResult`，Java 后端通过注册表路由，无需改核心逻辑。

#### ② 可编排 AI 管线（Pipeline DAG）

```
PipelineDefinition（JSON/YAML 配置）
  ├── steps: [summary, article, card, xiaohongshu, ...]
  ├── dependencies: { article: [summary], card: [summary] }
  └── prompts: { summary: "prompt_id_or_inline" }
```

管线定义与执行引擎解耦，新增步骤无需改引擎代码，用户/管理员可通过配置定制管线。

#### ③ 分发总线（Distribution Bus）

```
DistributionChannel (interface)
  ├── FeishuChannel      # v1.3 逻辑迁移
  ├── NotionChannel      # v1.3 逻辑迁移
  ├── WebhookChannel     # 通用 HTTP 回调
  └── EmailChannel       # 未来扩展
```

用户配置"生成完成后自动推送到哪些渠道"，系统通过事件触发，无需前端主动发起。

#### ④ 开放 REST API

为外部集成提供标准 API：

```
POST /open/v1/tasks          # 提交任务
GET  /open/v1/tasks/{id}     # 查询状态
GET  /open/v1/tasks/{id}/results  # 获取结果
```

配套 API Key 管理，支持外部工具（浏览器插件、CLI、第三方集成）调用。

**交付标准：** 接入一个新视频平台只需新增一个 Adapter 类，无需改其他代码；新增一种分发目标同理。

## v2.1 — 多平台版

**版本目标：** 在 v2.0 新架构上快速接入更多视频来源。

| 功能 | 说明 |
|------|------|
| YouTube Adapter | 通过 yt-dlp 提取字幕，自动识别语言 |
| 本地字幕上传 | 上传 .srt / .txt 文件直接处理，无需视频链接 |
| 抖音/西瓜 Adapter | 国内短视频平台字幕提取 |
| 多语言字幕支持 | 英文字幕自动翻译后再走管线 |

**版本目标：** 团队共享资源，从个人工具升级为团队工作台。
